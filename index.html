XPCC Cross Platform Component Communication

The XPCC package is a packet orientated communication library, that enables a transparent communication among components on PCs and microcontrollers. It provides an interface to communicate via CAN, TCP/IP and TIPC.

C++ and Microcontrollers

The library is written in C++. This gives a lot of advantages in terms of flexibility and usability. For microcontrollers, where memory is still short, we use optimized parts. For example the standard containers known from the STL are downsized to be used. The library also provides common interfaces to most low-level protocols, like UART, SPI and I2C, and classes for platform specific features like DMA and Timer/Counters.

The library supports the 8bit AVR platform, with ATmega and ATxmega targets, as well as the 32bit ARM Cortex-M0/M3/M4 platform, with emphasis on STM32 and LPC targets.

Elements of the Framework

  • C++ for AVRs and ARMs
    • new- and delete-operators
    • STL-container
    • IO-stream interface to hardware (console, RE232, CAN, I2C)
    • Abstracted hardware
  • Communication interface to:
    • CAN
    • TIPC
    • UART, SPI, I2C
  • Useful mathematical algorithm optimized for microcontrollers
  • Debug/log system (using the comunication interface)
  • Lightweight unit testing system
  • Drivers for a lot of external hardware

The History of this Framework

During the past seven years the Roboterclub Aachen e.V. has developed a software library for communication among components that are distributed on PCs and Microcontrollers. The primary aim has been the usage of this system in autonomous robots for the EUROBOT competition.
