XPCC Cross Platform Component Communication

The XPCC package is a packet orientated communication library, that enables a transparent communication among components on PCs and microcontrollers. It provides an interface to communicate via CAN, TCP/IP and TIPC.

C++ and Microcontrollers

The library is written in C++. This gives a lot of advantages in terms of flexibility and usability. For microcontrollers, where memory is still short, we use optimized parts. For example the standard containers known from the STL are downsized to be used. So this 'communication' library offers a lot more than only communication.

Elements of the Framework

  • C++ for AVRs
    • new- and delete-operators
    • STL-container
    • IO-stream interface to hardware (console, RE232, CAN, I2C)
  • Communication interface to:
    • CAN (via AT90CANxxx or MCP2515)
    • TIPC
  • Useful mathematical algorithm optimized for microcontrollers
  • Debug/log system (using the comunication interface)
  • Lightweight unit testing system
  • Drivers for various external hardware

The History of this Framework

During the last five years the Roboterclub Aachen e.V. has developed a software library for communication among components that are distributed on PCs and Microcontrollers. The primary aim has been the usage of this system in autonomous robots for the EUROBOT competition.
