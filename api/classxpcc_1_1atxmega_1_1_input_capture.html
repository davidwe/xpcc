Detailed Description

This template takes care of configuring the Pin, the Timer and the Event System for all three Input Capture Modes. The result is read and buffered independently to make sure no Error condition is encountered. The Timer Channel Interrupt is handled automatically, but may manually be overridden.

Template Parameters:
TIMERan Interrupt Timer/Counter 0 or 1 to configure and use.
PINany GPIO Input pin at which the signal is connected

This methods sets up the Event System to route the Pin Interrupt via the specified Multiplexer Channel to the Timer Channel and configures the Timer into the specified Input Capture Mode. Frequency Capture requires a rising edge trigger, Pulse Width Capture requires triggering on both edges (rising then falling edge). You can invert this by configuring the pin as inverted. In these two modes the Input Sense is automatically overwritten.

If you call this with constants, the compiler does all the calculations for the settings, not the Xmega.
