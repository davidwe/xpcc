3.5 Configuration files

Organization of the source

The 'project.cfg' and 'build.cfg' files

These files describe how to build a specific folder. They are simple INI-files and can be edited with any editor you like.

The configfiles are parsed with the module ConfigParser of python, but it also may change in future, so do't rely on it's specific syntax extensions.

build.cfg

These files have two sections: build contains all information needed to build the folder (and its subfolders), in defines you can list defines (together with default values for them) needed by files in this folder.

Example: 

[build]
target = atmega|atxmega

[defines]
mcp2515_clock = 16000000

The target string is a regular expression matched against the architecture string, generated by our SCons environment. This string has the form "architecture/device".

Example: 

ATMega644     => atmega/atmega644
ATXMega32A4   => atxmega/atxmega32a4
PC (Windows)  => pc/windows
PC (Linux)    => pc/linux
PC (Mac OS)   => pc/darwin

To match any AVR use 'atmega|atxmega', for PC only target 'pc'.

project.cfg

To build a directory as a standalone project just put a project.cfg file in it. In this are information about the desired target enviroment.

A SConstruct file then be generated automatically which will include all the source files from the folder and its subfolder to form the executable.

Example from the tests folder: 

[general]
name = raw_tipc_transmitter

[build]
architecture = pc

project.cfg

  • general
    • name
  • scons
    • regenerate = yes|no
  • build
    • architecture = pc|atmega|atxmega|arm7tdmi
    • device
    • clock
    • library_name
    • buildpath
  • environment options defined here will be appent to the scons environment by the xpcc tool
  • defines
  • program
    • tool = avrdude
    • port
  • fusebits
    • lfuse
    • hfuse
    • efuse

See section 4.1 Create a project using SCons for complete examples.
