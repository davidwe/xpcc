void xpcc::at91::PwmChannel::configure ( Clock  clock,
uint32_t  period,
uint32_t  configurationMask 
)

If the period is left aligned the counter counts from 0 to period and is reset. The resulting PWM frequency is:

 (Prescaler * period) / f_cpu
 \code
 
 If the period is center aligned the counter counts upwards to
 \c period and than downwards back to zero. Therefore the
 resulting PWM frequency is:
 
 \code
 (2 * Prescaler * period) / f_cpu

The internal counter is reset to 0 zero by this function.

void xpcc::at91::PwmChannel::setDutyCycle ( uint32_t  dutyCycle) [inline]

The duty cycle is updated at the next start of a period.

From the Errata Sheet: Setting Channel Duty Cycle at 0 in center aligned mode or at 0 or 1 in left aligned mode may change the polarity of the signal.

Problem Fix/Workaround:

  • Do not set PWM_CDTYx at 0 in center aligned mode.
  • Do not set PWM_CDTYx at 0 or 1 in left aligned mode.
Parameters:
dutyCycleMust be in range 1-2 to period (set during configure())
